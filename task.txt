Role and Objective:
You are required to build a "Cricbuzz-style" college sports tournament tracking web application. The focus is on a highly polished, read-only viewer frontend with "grand" UI elements, powered by a secure, robust admin dashboard for manual data entry.

Tech Stack: Make a simple full stack website to be deployed on vercel. It is a strict requirement of the porject to use mongodb for database.

Deployment context: Vercel

Application Architecture & Features:

Event Hierarchy: The highest level is an "Event" (e.g., General Championship 2026). An Event contains multiple "Sports" (e.g., Basketball, Chess). Each Sport has its own separate "Tournament."

Team & Squad Logic: "Teams" (e.g., Year 1, Year 2) register at the Event level. By default, they participate in all sports, but have the option to opt-out of specific sports. For every sport a Team participates in, they must register a specific "Squad" (roster of players).

Tournament Formats: Tournaments can be Knockout, Round Robin, Double Round Robin, or Hybrid (e.g., Group Stage Round Robin -> Knockout for qualifiers).

Bracketing/Grouping: During group stages, Admins must have the option to assign Squads to groups randomly or manually.

Live Admin Updates: Admins need a frictionless, highly efficient interface to manually update live scores and match statuses.

Viewer Experience: Dedicated tabs for each sport showing Live, Upcoming, and Past matches. Clicking any match reveals its detailed Match History/Timeline.

MongoDB Schema & Architecture (NoSQL Strategy)

Draft the Mongoose schemas for the following collections: User (Admins), Event, Sport, Team, Player, Squad, and Match.

Create a clean, functional layout for the Admin Dashboard with a sidebar navigation for: Events, Teams, Sports, Tournaments, and Live Matches.

Build the UI and Server Actions to create and manage an Event document in MongoDB.

Build the UI to add Teams to the Event.

Build the UI to add Sports to the Event and configure the tournament format (Dropdown: Knockout, Round Robin, etc.).

Build the interface where a Team registers a Squad (selecting Players) for a specific Sport. Include the logic to let a Team opt-out of a specific sport.

PHASE 4: Tournament Generation & Bracketing Logic (Admin)

Build the UI for the selected tournament format based on the approved logic.

If Group Stage/Round Robin is selected, provide an interface to create Groups (e.g., Group A, Group B) and assign Squads either randomly (via a randomize function) or by manual drag-and-drop selection.

Build the Server Actions to auto-generate the Match documents in MongoDB based on the format.

Allow Admins to manually set the Date and Time for these generated matches.

PHASE 5: Live Match Update Interface (Admin)
Create a dedicated "Live Desk" dashboard for Admins.

It should fetch Match documents with status = 'Live'.

Build a fast, simple form allowing the Admin to increment scores, push a new event object into the embedded MatchEvents array (e.g., "Player X scored a 3-pointer", "Yellow Card"), and change match status to 'Completed'.

PHASE 6: The Viewer UI - The "Cricbuzz" Frontend
Build the public-facing application wrapper. Implement a dark/light mode toggle.

Build the Sport Navigation Tabs. When a sport is clicked, dynamically route to that sport's dashboard.

Inside a sport, create three distinct tabs: Live, Upcoming, and Past.

Design the Scorecard component. For 'Live' matches, include a CSS-animated pulsing red dot. For 'Upcoming' matches, use date-fns to show relative time (e.g., "Starts in 2 hours").

You are allowed to do design compromises of your own.
Don't use any paid tools or paid APIs for development and also create a walkthrough on how these can be deployed live
